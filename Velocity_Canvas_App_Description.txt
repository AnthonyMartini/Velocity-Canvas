# Velocity Canvas: Application Overview (Start to Finish)

## 1. Introduction
Velocity Canvas is a high-performance, AI-driven development environment designed specifically for Power Apps creators. Its primary mission is to **mimic the look and feel of Microsoft Power Apps**, providing a familiar and intuitive experience for developers. By bridging the gap between natural language descriptions and production-ready Power Apps YAML (pa.yaml), it allows users to rapidly prototype UI components in an environment that feels like home, then seamlessly export their work directly into Canvas Studio.

## 2. Core Architecture
The application is built using a modern decoupled architecture:
- **Frontend Framework**: React (Vite-based) with Tailwind CSS for layout and Vanilla CSS for precise component styling.
- **Component Schema System**: A centralized repository of JSON schemas defines every supported Power Apps control, including their default properties and metadata.
- **Formula Engine**: A custom Abstract Syntax Tree (AST) parser and evaluator (`FormulaParser.jsx`) that enables live PowerFx-style logic within the browser.
- **UI State Management**: React state handles a complex hierarchical tree of components, allowing for deep nesting and instant reactivity.

## 3. Supported Components and Properties

### A. Core Controls
| Component | Description | Key Properties |
| :--- | :--- | :--- |
| **Button** | Interactive click target | `Text`, `OnSelect`, `Fill`, `Color`, `Radius`, `HoverFill` |
| **Label** | Text display control | `Text`, `Align`, `VerticalAlign`, `Color`, `Size`, `FontWeight` |
| **TextInput** | User text entry field | `Default`, `HintText`, `Mode` (Single/Multi/Password), `OnChange` |
| **Rectangle** | Visual shape / Background | `Fill`, `OnSelect`, `Visible`, `FocusedBorder` |
| **Icon** | Vector-based SVG icons | `Icon` (selector), `Rotation`, `Color`, `OnSelect` |
| **HtmlText** | Rich text / HTML renderer | `HtmlText`, `Padding`, `Border`, `Fill` |

### B. Input & Data Controls
| Component | Description | Key Properties |
| :--- | :--- | :--- |
| **Dropdown** | List selection menu | `Items`, `Default`, `Selected`, `AllowEmptySelection` |
| **ComboBox** | Searchable multi-select | `Items`, `SearchFields`, `DisplayFields`, `SelectMultiple` |
| **Checkbox** | Two-state toggle | `Text`, `Default` (Boolean), `OnCheck`, `OnUncheck` |
| **DatePicker** | Calendar date selector | `DefaultDate`, `Format`, `StartYear`, `EndYear`, `IsEditable` |

### C. Layout & Container Controls
| Component | Description | Key Properties |
| :--- | :--- | :--- |
| **Container** | Logical grouping of children | `DropShadow`, `Border`, `Radius`, `children` (Collection) |
| **Gallery** | Dynamic list of items | `Items`, `TemplateSize`, `Layout` (Vertical/Horizontal), `WrapCount` |

### D. System Nodes
| Node | Description | Properties |
| :--- | :--- | :--- |
| **App** | Root application node | `OnStart`, `Theme` (Internal) |
| **Screen** | Individual app surface | `Fill`, `Width`, `Height` |

### E. Universal Properties
Most components share these standard Power Apps properties:
- **Position & Size**: `X`, `Y`, `Width`, `Height`
- **Visibility**: `Visible` (Boolean)
- **State**: `DisplayMode` (Edit, View, Disabled)
- **Styling**: `Fill`, `Color`, `BorderColor`, `BorderStyle`, `BorderThickness`
- **Internal**: `id`, `name`, `type`, `parent`

### C. Design Philosophy
The application is intentionally designed to **mimic the Power Apps Studio environment**. This ensures that users can transition from the Velocity Canvas to the official Power Apps environment with zero friction. The iconography, color palette, and hierarchical workflows are all built to feel native to a Power Apps developer.

## 4. Key Features
- **AI Component Generator**: Converts natural language prompts to UI trees.
- **Visual Canvas Editor**: Real-time drag-and-drop and property manipulation.
- **PowerFx logic engine**: Live evaluation of formulas and variables.
- **Global Error Tracking**: Centralized validation of every property in the app.
- **Documentation Library**: Integrated reference for all components and enums.

## 5. The User Journey
1. **Describe**: Enter a prompt (e.g., "A header with a search bar").
2. **Generate**: AI builds the initial schema and visual preview.
3. **Refine**: Manually adjust properties or add logic in the Editor.
4. **Validate**: Fix any errors identified by the engine.
5. **Export**: Copy the production-ready YAML into Power Apps.
